# @dreamworld/vault

A secure data storage library with encrypted storage capabilities for web applications — supports both AES-256-GCM encrypted (secure) and plaintext (insecure) modes, with password-based locking, auto-lock on inactivity, and cross-tab state synchronization.

---

## 1. User Guide

### Installation & Setup

```bash
yarn add @dreamworld/vault
```

This package is ESM-only (`"type": "module"` in `package.json`). It requires a browser environment with access to `localStorage`, `sessionStorage`, and the Web Crypto API (`crypto.subtle`).

**Development server** (from `package.json` scripts):

```bash
yarn start   # launches @web/dev-server, opens /demo/ with --watch
```

---

### Basic Usage

```javascript
import Vault from '@dreamworld/vault';

// Create and auto-initialize from persisted state
const vault = new Vault();

// --- Insecure mode (default): plaintext storage ---
vault.store('theme', 'dark');
vault.get('theme');      // 'dark'
vault.hasKey('theme');   // true
vault.isEmpty();         // false
vault.remove('theme');

// --- Objects are supported ---
vault.store('prefs', { lang: 'en', fontSize: 14 });
vault.get('prefs');      // { lang: 'en', fontSize: 14 }
```

---

### Secure Mode: Encrypt, Lock & Unlock

The `keys` object maps an auth-provider name to a Base64URL-encoded JSON blob containing PBKDF2-SHA256 KDF parameters and an AES-GCM-encrypted private key. This blob is **produced externally** (not generated by this library). See [Advanced Usage](#advanced-usage) for the blob schema.

```javascript
import Vault from '@dreamworld/vault';

const vault = new Vault();

// 1. Make vault secure (encrypts any existing data)
const keys = {
  password: '<base64url-encoded-key-blob>'
};
await vault.secure(keys, 'password', 'my-strong-passphrase');
// vault is now secure AND unlocked

// 2. Store encrypted data
vault.store('apiKey', 'sk-abc123');
vault.store('config', { endpoint: 'https://api.example.com' });

// 3. Lock the vault (clears in-memory decrypted data)
await vault.lock();

// 4. Unlock the vault (decrypts all data back into memory)
await vault.unlock('password', 'my-strong-passphrase');

// 5. Read data
vault.get('apiKey');   // 'sk-abc123'

// 6. Lock again
await vault.lock();
```

---

### Events

```javascript
vault.on('unlock',  () => console.log('Vault unlocked'));
vault.on('lock',    () => console.log('Vault locked'));
vault.on('secure',  () => console.log('Vault is now secure'));
vault.on('insecure',() => console.log('Vault is now insecure'));
vault.on('destroy', () => console.log('Vault destroyed'));
```

### API Reference

#### `Vault` Class — `vault.js`

**Constructor**

| Signature | Description |
|---|---|
| `new Vault()` | Creates a vault instance and initializes from persisted `localStorage`/`SessionStorage` state. Sets up cross-tab sync and auto-lock listeners. |

---

**Public Methods**

| Method | Parameters | Returns | Throws | Description |
|---|---|---|---|---|
| `unlock(keyName, passcode)` | `keyName: String`, `passcode: String` | `Promise<Boolean>` | `{ code: 'INVALID_PASSCODE', message: 'Invalid passcode' }` | Derives private key from passcode, decrypts all stored data into session memory, stores unlock state for cross-tab sync. Emits `'unlock'`. |
| `store(key, value)` | `key: String`, `value: String \| Object` | `void` | `Error` if vault is secure and locked | Stores value in session memory and `localStorage`. Encrypts before persisting if vault is secure. Objects are `JSON.stringify`-ed. |
| `get(key)` | `key: String` | `String \| Object \| undefined` | `Error` if vault is secure and locked | Retrieves value from session memory. Returns `undefined` if key not found. |
| `remove(key)` | `key: String` | `void` | `Error` if vault is secure and locked | Removes value from both session memory and `localStorage`. |
| `secure(keys, keyName, passcode)` | `keys: Object`, `keyName: String`, `passcode: String` | `Promise<void>` | `Error` if vault is already secure | Encrypts all existing data with derived key, persists `keys`, stores unlock state. Emits `'secure'`. |
| `changeKeys(keys)` | `keys: Object` | `Promise<void>` | — | Replaces the stored encryption key blobs. Works in both locked and unlocked states. No-op if vault is not secure. |
| `changeSettings(settings)` | `settings: Object` | `Promise<void>` | — | Updates and persists vault settings. Works in both locked and unlocked states. No-op if vault is not secure. Re-applies auto-lock configuration. |
| `insecure()` | — | `Promise<void>` | `Error` if vault is locked | Decrypts all data to plaintext in `localStorage`, removes all keys. Emits `'insecure'`. |
| `lock()` | — | `Promise<void>` | `Error` if vault is not secure | Clears all decrypted data from session memory. Emits `'lock'`. |
| `destroy()` | — | `Promise<void>` | — | Clears all vault data from `localStorage` and session memory. Resets internal state. Emits `'destroy'`. Vault must be re-created after this call. |
| `isSecure()` | — | `Boolean` | — | Returns `true` if the vault has encryption keys stored. |
| `isLocked()` | — | `Boolean` | — | Returns `true` if vault is secure and the private key is not in memory. Always `false` for insecure vaults. |
| `hasKey(key)` | `key: String` | `Boolean` | `Error` if vault is secure and locked | Returns `true` if a value is stored under the given key. |
| `isEmpty()` | — | `Boolean` | `Error` if vault is secure and locked | Returns `true` if no data values are stored. |

**Configuration Options**

Passed to `changeSettings(settings)`. Stored in `localStorage` under `vault_settings`.

| Property | Type | Default | Description |
|---|---|---|---|
| `autoLockTimeout` | `Number` | `0` | Minutes of user inactivity before automatic lock. `0` disables auto-lock. Activity events tracked: `mousedown`, `mousemove`, `keypress`, `scroll`, `touchstart`, `click`. |

#### `utils.js` — Exported Cryptographic Functions

These are low-level utilities used internally by `Vault`. They are also exported for consumers who need direct access to the encryption primitives (e.g., for generating key blobs).

| Function | Parameters | Returns | Description |
|---|---|---|---|
| `getPrivateKeyFromPasscode(authProvider, passcode, privateKeyByAuths)` | `authProvider: String`, `passcode: String`, `privateKeyByAuths: Object` | `Promise<CryptoKey>` | Derives a 256-bit AES-GCM key via PBKDF2-SHA256 (100,000 iterations), then uses it to decrypt the stored private key blob. Returns a `CryptoKey` ready for `encrypt`/`decrypt`. Throws if `authProvider` not found or KDF is not `pbkdf2-sha256`. |
| `encrypt(key, plaintext)` | `key: CryptoKey`, `plaintext: String` | `Promise<String>` | Encrypts `plaintext` with AES-GCM using a random 12-byte IV. Returns payload in format: `base64(IV)$$base64(ciphertext)`. |
| `decrypt(key, payload)` | `key: CryptoKey`, `payload: String` | `Promise<String>` | Decrypts a `base64(IV)$$base64(ciphertext)` payload. Returns the original plaintext string. |
| `getBase64PrivateKey(key)` | `key: CryptoKey` | `Promise<String>` | Exports a `CryptoKey` to its raw bytes and encodes as Base64. Used to persist unlock state in session storage. |
| `privateKeyFromBase64(b64Key)` | `b64Key: String` | `Promise<CryptoKey>` | Imports a Base64 raw-bytes string as an AES-GCM `CryptoKey` with `encrypt`/`decrypt` usage. |

### Advanced Usage

#### Cross-Tab Synchronization

The vault uses `@dreamworld/session-storage` (a cross-tab in-memory store) to propagate state changes across browser tabs:

- **Unlock state**: When one tab calls `unlock()`, all other tabs detect the new `vault_unlockPrivateKey` entry in session storage, derive the key, and emit `'unlock'` locally.
- **Lock state**: When one tab calls `lock()`, the session key is removed; other tabs detect this and emit `'lock'` locally.
- **Activity tracking**: When auto-lock is enabled, user activity in any tab resets the inactivity timer across all tabs via a shared `vault___activity__` session key.

#### Key Blob Format (`keys` object)

The `keys` parameter in `secure(keys, keyName, passcode)` and `changeKeys(keys)` must follow this schema:

```javascript
const keys = {
  // authProvider name (e.g. 'password', 'google', 'github')
  password: '<base64url-encoded-JSON-blob>'
};
```

The JSON blob (before Base64URL encoding) must conform to:

```json
{
  "kdf": "pbkdf2-sha256",
  "params": {
    "salt": "<base64url>",
    "iter": 100000,
    "iv":   "<base64url>"
  },
  "ct": "<base64url>"
}
```

Where:
- `salt` — PBKDF2 salt used during key derivation
- `iter` — PBKDF2 iteration count (must be `100000` as enforced by `utils.js`)
- `iv` — AES-GCM IV used to encrypt the raw private key bytes
- `ct` — AES-GCM ciphertext of the 32-byte raw private key

This blob is **not generated by this library**. It must be produced externally (e.g., during user account setup or credential provisioning).

#### Auto-Lock

```javascript
const vault = new Vault();

// Enable auto-lock after 5 minutes of inactivity
await vault.changeSettings({ autoLockTimeout: 5 });

vault.on('lock', () => {
  // Prompt user to re-enter passcode
});
```

## 2. Developer Guide / Architecture

### Architecture Overview

**Design patterns:**

- **Observer (EventEmitter)**: `Vault` extends `@dreamworld/event-emitter`, emitting lifecycle events (`unlock`, `lock`, `secure`, `insecure`, `destroy`) that consumers subscribe to.
- **Dual-Storage Strategy**: A two-layer storage model separates persistence from in-memory access:
  - `localStorage` — persistent store for encrypted/plaintext data, keys, and settings
  - `@dreamworld/session-storage` — cross-tab in-memory store for decrypted data and unlock state
- **Web Crypto API**: All cryptographic operations (`PBKDF2`, `AES-GCM`) use `crypto.subtle` — the browser's native, hardware-accelerated cryptography API. No third-party crypto library is used.

**Module responsibilities:**

| Module | Role |
|---|---|
| `vault.js` | Orchestrates storage, encryption lifecycle, auto-lock, and cross-tab sync. Exposes the public API. |
| `utils.js` | Stateless cryptographic primitives: key derivation, encryption, decryption, key serialization. |

### Storage Layout

All keys in `localStorage` and session storage are namespaced under the `vault_` prefix:

| Key | Store | Contents |
|---|---|---|
| `vault_keys` | `localStorage` + session | JSON object `{ [authProvider]: blob }` |
| `vault_settings` | `localStorage` + session | JSON object `{ autoLockTimeout: Number }` |
| `vault_initialized` | session only | Boolean flag; prevents re-initialization on new tab |
| `vault_unlockPrivateKey` | session only | Base64-encoded raw AES key bytes; cleared on lock |
| `vault___activity__` | session only | `Date.now()` timestamp; updated on user activity for cross-tab timer sync |
| `vault_data_<key>` | `localStorage` + session | Encrypted string (`IV$$ciphertext`) in secure mode; raw value in insecure mode |

### Encryption Flow

```
Passcode
   │
   ▼
PBKDF2-SHA256 (100,000 iter, random salt)
   │
   ▼
AES-GCM-256 key  ──► Decrypt(stored ct)  ──► Raw 32-byte private key
                                                      │
                                                      ▼
                                             Import as AES-GCM CryptoKey
                                                      │
                                           ┌──────────┴──────────┐
                                           │                     │
                                        encrypt()            decrypt()
                                     random 12-byte IV     IV from payload
                                        AES-GCM-256         AES-GCM-256
                                           │
                                           ▼
                               "base64(IV)$$base64(ciphertext)"
                                  stored in localStorage
```

### Initialization Sequence

```
new Vault()
   │
   ├─ Check session: vault_initialized?
   │     YES → _syncUnlockPrivateKey() → done
   │     NO  ↓
   │
   ├─ Load keys + settings from localStorage
   ├─ Merge settings with defaultSettings
   ├─ Write keys + settings to session storage
   ├─ Load plain (insecure) data to session storage
   ├─ _syncUnlockPrivateKey()  ← restores unlock state if session key present
   └─ Set vault_initialized = true in session
```
